## Что строим — Dota 2 AI Match Analyzer

**Идея:** Telegram-бот, который разбирает матчи Dota 2 и даёт **конкретные** советы по ошибкам, а не пустые цифры типа KDA.

**Целевая аудитория:** игроки 3-5k MMR, которые хотят расти. Они готовы платить за инструмент, который реально помогает.

**Пример вывода для пользователя:**
> Матч 7891234567, проиграли за 38 минут.
>
> **Главные ошибки:**
> — **14:32** — ты ушёл в лес вместо отжима T1 на верху. Ваша команда выиграла файт за минуту до этого, башня была доступна. Потеря темпа ~2200 золота.
> — **22:10** — погиб в незаваренной зоне у дальнего лагеря Радиант. Это уже третья смерть в одной и той же точке за матч.
> — **Опоздал с BKB** на 4 минуты против Lina+Lion. Купил в 26:00, нормально для твоего героя — 22:00.

## Архитектурные решения (приняты в прошлом чате)

Запиши их и не пересогласовывай:

1. **AI-движок для генерации советов:** **Claude Haiku API** (`claude-haiku-4-5-20251001`). Дёшево (~0.5-1 рубль за разбор), быстро, качественно. Получим API-ключ через https://console.anthropic.com.

2. **Подход к старту:** **Сначала PoC-скрипт** (как с `skin_checker.py`). Простой Python-файл, который принимает Match ID, тянет данные из OpenDota и печатает в консоль готовый разбор. Telegram-бота строим только после того, как анализ работает на сырых данных.

3. **Способ ввода матча:** **Steam ID + автозабор последнего матча**. Пользователь один раз привязывает свой Steam ID, дальше команда `/analyze` всегда берёт его последнюю игру. Опция указать конкретный Match ID — добавим позже как доп.

## Архитектура (3 слоя)

```
[OpenDota API] → [Сборщик данных] → [Аналитический движок] → [LLM-генератор советов] → [пользователь]
                       ↓                      ↓                         ↓
                   raw JSON           структурированные             текст на русском
                                          "ошибки"
```

1. **Сборщик данных** — берёт matchID, тянет полную статистику из OpenDota API (общая инфа, тайминги, события покупок, бенчмарки).
2. **Аналитический движок** — преобразует сырые данные в список структурированных "ошибок" по правилам. Например: "опоздал с BKB на N минут", "умер в зоне X без обзора", "GPM на 30% ниже бенчмарка героя".
3. **LLM-генератор** (Claude Haiku) — превращает список ошибок в живой текст-разбор на русском, с приоритизацией (топ-3 самых критичных).

## Следующий шаг (с чего начинать в новом чате)

**Конкретно первый PoC-скрипт `dota_analyzer.py`:**

1. Принимает hardcoded Match ID (Pavel вставит ID одного из своих недавних матчей).
2. Делает GET-запрос к `https://api.opendota.com/api/matches/{match_id}`.
3. Достаёт из JSON-ответа базовую инфу: герой Pavel'а, KDA, GPM/XPM, длительность, результат, бенчмарки.
